# Work-order Summary

## Batch objective
Execute the UI/UX revamp as bounded slices that preserve the current authority model, extend the current operator surface, and produce a cockpit-grade experience instead of a loose collection of panels.

## Phase sequence
- Discover: WO-UX-001
- Define: WO-UX-002
- Visualize: WO-UX-003
- Architect: WO-UX-004, WO-UX-005
- Implement: WO-UX-006 through WO-UX-013
- Verify: WO-UX-014
- Release / Improve: WO-UX-015

## Critical approval gates
- WO-UX-002
- WO-UX-003
- WO-UX-004
- WO-UX-005
- WO-UX-013

These are the authority-lock and platform-fit decisions that should be reviewed before implementation fans out.

## Highest-value implementation slices
1. Cockpit shell and saved layouts
2. Graph interaction upgrades
3. Inspector and drill-down chain
4. Command palette, quick actions, and chat suggestions
5. Plan/session convergence
6. Motion and token system

## Release gate themes
- state-authority clarity
- accessibility and reduced motion
- graph performance
- safe launch / recovery
- evidence and learning completeness
