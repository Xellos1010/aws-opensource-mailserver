# Architect Brief

## Mission
Preserve the current Foundry authority model while making the operator experience significantly faster and clearer.

## Primary architecture rules
- One write-capable desktop cockpit.
- Projection surfaces remain read-first.
- Object navigation over menu navigation.
- Stable drill-down chain.
- Declared vs observed state visibly separated.
- Evidence and verifier posture are first-class.
- Desktop and mobile are not false peers.

## Review focus
- Do the ADRs preserve authority correctly?
- Does the shell layout expose the right operator questions?
- Are graph, board, timeline, diff, and evidence mapped to real decisions?
- Are quick actions and chat safe?
- Is accessibility a release gate?
