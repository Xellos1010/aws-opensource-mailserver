# Knowledge Base Blueprint

## Why multiple knowledge bases
A UI/UX planning agent should not retrieve everything from one giant bucket. It needs:
- repo authority and contracts,
- operator state and runtime projections,
- official design standards,
- visualization patterns,
- user task corpus,
- telemetry/evidence/learning,
- platform adaptation guidance.

## Existing foundation
The repo already has a foundational docs knowledge base with 1124 documents mapped to domain packs. This is the seed layer, not the full UI/UX diagnosis layer.

## Recommended UI/UX knowledge bases
1. Repo Authority & Contracts
2. Operator Surface Runtime Projections
3. UI/UX Standards — Web, iOS, Android
4. Visualization Pattern Library
5. User Task & Decision Corpus
6. Telemetry, Evidence & Learning
7. Platform Adaptation

## Retrieval guidance
- Critical pack freshness window: 14 days
- Important pack freshness window: 30 days
- Never let RAG override artifact authority
- Mark every retrieved item with provenance and freshness
