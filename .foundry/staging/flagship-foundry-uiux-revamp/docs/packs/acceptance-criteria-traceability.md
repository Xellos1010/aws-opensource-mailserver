# Acceptance Criteria Traceability

| User pain / desired outcome | Architecture response | Work order(s) |
|---|---|---|
| Too many menus | Object-based navigation + fixed shell | WO-UX-002, WO-UX-003, WO-UX-007 |
| Graph is not interactive enough | Map-centric navigation, overlays, focus path, compare, minimap | WO-UX-008 |
| Need quick actions by intent | Command palette + quick-action resolver | WO-UX-001, WO-UX-010 |
| Need panel control | Dockable shell + saved layouts | WO-UX-007 |
| Need drill-down from system to code/evidence | Explicit drill-down chain + inspector tabs | WO-UX-005, WO-UX-009 |
| Need chat interface for orchestration | Chat suggestion layer with approval-safe actions | WO-UX-010 |
| Need Trello/Jira-like coordination | Plan board and session board convergence | WO-UX-011 |
| Need cross-platform adaptation | Mobile triage surfaces + platform split | WO-UX-013 |
| Need engaging, intuitive transitions and controls | Motion/toggle/accessibility baseline | WO-UX-012, WO-UX-014 |
